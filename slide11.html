JSON Storage Structure

Data persists in users.json. Each user object contains a unique UUID, email, bcrypt password hash, and role (admin/user). This serves as a lightweight NoSQL-style document store.

Core Operations (CRUD)

Implemented service layer functions: findByEmail() for auth, addUser() for registration, and utility functions to parse/stringify JSON data dynamically.

Concurrency & Integrity

Addressed file locking issues using Atomic Writes: writing to a temporary file first, then renaming it to ensure data isn't corrupted during simultaneous requests.

Scalability & Migration

While excellent for rapid prototyping (Zero config), it lacks ACID compliance. Future Plan: Migrate to MySQL using seeding scripts to transfer existing JSON data to relational tables.
